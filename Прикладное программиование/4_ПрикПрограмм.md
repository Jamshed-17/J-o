
1. Как в реляционной модели данных определяется понятие «отношение»? В чем разница между реляционным отношением и таблицей?
Ответ: Отношение – это множество кортежей, имеющих один и тот же набор атрибутов. Атрибут – это пара вида ‹Имя_атрибута: Имя_домена›. Домен – это множество допустимых значений. Реляционное отношение – это абстрактное понятие, а таблица – это его конкретное представление. <...> Реляционное отношение представляет собой множество, поэтому оно не может содержать два одинаковых кортежа, и порядок кортежей в отношении не имеет значения. В таблице же строки могут дублироваться, и они всегда располагаются в определенном порядке
    
2. Что такое потенциальный ключ? Поясните разницу между естественными и суррогатными ключами.
Ответ: Потенциальный ключ – это минимальный набор атрибутов, однозначно идентифицирующий любой кортеж отношения. Минимальность означает, что удаление любого атрибута из набора нарушает уникальность. Если потенциальный ключ образован атрибутами, которые имеют самостоятельный смысл в предметной области (например, Номер паспорта), то его называют **естественным ключом**. Если же атрибут был специально введен в отношение для целей идентификации и не имеет самостоятельного смысла (например, автоматически генерируемый ID), то такой ключ называется **суррогатным**
    
3. Какие реляционные отношения являются совместимыми по типу?
Ответ: Два отношения являются **совместимыми по типу** (или совместимыми по объединению), если они имеют одинаковые заголовки, то есть одно и то же количество атрибутов, причем соответствующие атрибуты определены на одних и тех же доменах
	
4. Перечислите и дайте определения реляционных операций, входящих в начальную алгебру Кодда
Ответ: Начальная реляционная алгебра, предложенная Э. Коддом, включает восемь операций:

- **Выбор (σ)** – формирует новое отношение, содержащее те кортежи исходного отношения, которые удовлетворяют заданному условию.
    
- **Проекция (π)** – формирует новое отношение, содержащее указанное подмножество атрибутов исходного отношения с удалением дубликатов.
    
- **Объединение (R ∪ S)** – формирует отношение, содержащее все кортежи, которые принадлежат либо R, либо S, либо обоим отношениям. Отношения должны быть совместимы по типу.
    
- **Пересечение (R ∩ S)** – формирует отношение, содержащее все кортежи, принадлежащие одновременно и R, и S. Отношения должны быть совместимы по типу.
    
- **Разность (R – S)** – формирует отношение, содержащее все кортежи, которые принадлежат R, но не принадлежат S. Отношения должны быть совместимы по типу.
    
- **Декартово произведение (R × S)** – формирует отношение, кортежи которого являются сцеплением каждого кортежа из R с каждым кортежем из S.
    
- **Соединение (R ⨝ S)** – формирует отношение, кортежи которого являются сцеплением кортежей из R и S, удовлетворяющих условию соединения. Частный случай – естественное соединение.
    
- **Деление (R ÷ S)** – операция, обратная декартову произведению. Результирующее отношение содержит такие значения кортежей из R, которые в сочетании с каждым кортежем из S присутствуют в R

5. Выражения реляционной алгебры позволяющие получить следующие данные:

а) список студентов (номер студенческого и фамилию с инициалами) группы 382

Ответ: π_StudID, FIO ( σ_Group='382' (St) )

б) список студентов, сдавших экзамен по физике на "пять"

Ответ: π_StudID, FIO ( St ⨝ ( σ_Subject='Физика' ∧ Result='5' (Res) ) )

в) список дисциплин, по которым все студенты, сдавшие эту дисциплину, получили только пятёрки

Ответ: π_Subject (Res) − π_Subject ( ( π_Subject, StudID (Res) − π_Subject, StudID (σ_Result='5' (Res)) ) )

г) список студентов, сдавших все упомянутые в Res учебные дисциплины на 5

Ответ: π_StudID, FIO ( St ⨝ ( π_StudID (Res) ÷ π_Subject (σ_Result='5' (Res)) ) )
