# РСМ ID
Чтобы реализовать возможность впоследствии сделать такие системы на разные фестивали, а также не заставлять постоянных участников регистрировать аккаунт каждый год заново - нужно сделать единый ID. За пользователем будет закреплён никнейм в формате "@Jamshed", который будет использоваться во всех дальнейших операциях. 
В этом ID будут следующие данные:
- Фамилия
- Имя 
- Отчество
- Учебное заведение
- Никнейм
- Логин
- Пароль
- Email
- Моб. телефон
- Фото

Я пока думаю как сделать верификацию пользователей, но в любом случае на следующих этапах их данные будут отредактированы (если они участвуют)

В личном кабинете этого ID можно будет посмотреть на каких фестивалях человек принимал участие (в том числе отдельно по каждому году) и скачать сертификат участника или диплом победителя в PDF

А ещё к этому ID можно будет подвязать телеграм, чтобы с ними связывался **бот**.

# Заполнение репертуарки
Один из самых важных этапов. Он упростит жизнь и нам и участникам. Проще подавать, модерировать и отслеживать.

1. **Умные поля.** Для разных случаев используется разное количество полей (например для медиа не нужны поля "Название номера" или "Автор музыки" и т.д.) здесь это решается ситуативным вводом. В зависимости от прошлого выбора ему открываются (или не открываются) следующие поля. 
2. **Типизация ответов**. Иногда в поле нужно не заполнять что-то своё, а выбрать из вариантов. Например категория (проф/не проф), или направление и номинация, тоже выбираются из определённого списка, сформированного для фестиваля этого года заранее.
3. **Каскадное указание фигурирующих людей**. Если есть руководитель, технический сопровождающий, или это не сольный номер - нужно указывать всех остальных людей. Здесь можно сделать так: каждый член команды, руководитель и техник регистрируются в РСМ ID и указывают учебное заведение; человек заполняющий репертуарку ищет их по никнейму, или фамилии, с помощью выборки по учебному заведению (то есть ему в вариантах предлагают людей только из его вуза/колледжа). Таким образом у нас появляется полная прозрачность, правдивая статистика и возможность формирования коллективов и номеров для дальнейшей работы.
4. **Валидация данных**. Можно сразу проверять песни на соответствие с РАО и ВОИС, впоследствии формируя отчёт без заморочек.
5. **Автоматизация**. Благодаря такому подходу можно избежать многочасовых проверок каждой заявки с фигово заполненными полями, потому что данные будут строго определённого вида. Если необходимость проверок останется - можно сделать удобный интерфейс, в котором не придётся всё скачивать, записывать в табличку и т.д. Просто нажать на кнопку "Одобрить" или "Отклонить" и указать причину. Если заявку отклонили пользователю приходит уведомление и он исправляет ошибки.

# Работа с документами
Работу с документами тоже можно значительно упростить. 
Момент спорный, можно от этого отказаться и дальше работать вручную, но решение с автоматизацией обеспечит значительное упрощение и ускорение работы. Я предлагаю полу-автоматический вариант.
